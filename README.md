# Реестр доверенных лиц — MVP

Владелец создаёт «капсулу» (текст + ссылка на видео), ведёт внутри неё список доверенных лиц по номерам телефонов и вручную открывает/закрывает доступ. У капсулы **одна публичная ссылка** `/capsule/{uuid}` — единая для всех. Посетитель вводит свой номер: если номер есть в списке капсулы — получает SMS-код и после верификации видит контент (когда доступ открыт); если номера нет — «Номер не найден в базе доступа», SMS не отправляется.

## Запуск

```bash
npm install
npm start
# → http://localhost:3000
```

`.env` уже создан с случайным `JWT_SECRET`. Пока `SMSRU_API_KEY` пуст — SMS-коды печатаются в консоль сервера (тестовый режим).

## Реальные SMS (SMS.ru)

Три режима — по возрастанию «настоящести»:

1. **Без ключа** (по умолчанию): SMS не отправляются, код печатается в консоль/лог сервера. Подходит для локальной разработки.
2. **Ключ + `SMSRU_TEST=1`**: запросы уходят в SMS.ru, но с флагом `test=1` — провайдер имитирует отправку, деньги не списываются, SMS не приходит. Проверка интеграции с API.
3. **Ключ + `SMSRU_TEST=0`**: боевая отправка.

Как получить ключ:
1. Зарегистрируйтесь на [sms.ru](https://sms.ru) (вход по своему номеру телефона).
2. `api_id` виден сразу в личном кабинете — скопируйте его в `SMSRU_API_KEY` в `.env`.
3. На свой подтверждённый номер SMS.ru позволяет слать бесплатно (лимит ~5 в день) — удобно для теста «на себя». Отправка на чужие номера — с баланса; для отправителя по умолчанию используется общее имя, своё имя отправителя требует регистрации у оператора.

Важно: для рассылки кодов физлицам в РФ у SMS.ru потребуется подтверждение аккаунта; альтернативы с таким же простым HTTP API — SMSC.ru, GreenSMS.

## Деплой (бесплатно)

Backend на Node.js нельзя захостить на GitHub Pages (там только статика), поэтому схема такая: код лежит на GitHub, а запускается на [Render](https://render.com) (бесплатный тариф):

1. Зайдите на render.com, войдите через GitHub.
2. New → Blueprint → выберите репозиторий `reestr-mvp` — Render прочитает `render.yaml` и настроит всё сам (JWT_SECRET сгенерирует автоматически).
3. После деплоя приложение получит адрес вида `https://reestr-mvp.onrender.com` с HTTPS из коробки.

Ограничения бесплатного тарифа: сервис засыпает после 15 минут простоя (первый запрос будит его ~30–50 сек), диск эфемерный — SQLite-база обнуляется при каждом деплое/рестарте. Для теста это приемлемо; для реальных пользователей с номерами граждан РФ хостинг должен быть в РФ (ФЗ-242): Selectel / VK Cloud / Yandex Cloud + Postgres.

## Стек

- **Backend**: Node.js 24 + Express, БД — SQLite через встроенный `node:sqlite` (файл `data/reestr.db`, создаётся автоматически). Схема совместима с Postgres.
- **Frontend**: статичные HTML/CSS/JS без фреймворков, mobile-first, на русском.
- Пароли — bcrypt; SMS-коды хранятся только как SHA-256 хэш; ссылки доступа — UUID v4; сессия доверенного лица — JWT на 20 минут.

## Страницы

| URL | Что это |
|---|---|
| `/` | Вход / регистрация владельца |
| `/dashboard.html` | Кабинет: капсулы, доверенные лица, журнал действий |
| `/capsule/{uuid}` | Публичная двухшаговая страница верификации (номер → SMS-код) |

## Ограничения и защита

- Номер проверяется по списку доверенных лиц капсулы; чужим номерам SMS не отправляется.
- SMS-код: 6 цифр, TTL 5 минут, максимум 5 попыток ввода → блокировка номера на 15 минут.
- Rate limit отправки: не чаще 1 SMS / 60 сек и не более 5 SMS / час на номер.
- Rate limit проверки номера: не более 10 попыток в час с одного IP.
- Ответ на проверку номера приходит с одинаковой задержкой (~0.8 с) независимо от результата — защита от определения списка номеров по времени ответа.
- Каждое событие (phone_check_fail/ok, otp_sent, otp_fail/ok, content_viewed) пишется в `audit_log` с IP, user-agent и номером — журнал виден владельцу в кабинете.
- Чекбоксы согласия на обработку ПДн при регистрации и при верификации (152-ФЗ).

## API (кратко)

- `POST /api/register`, `POST /api/login` — владелец (JWT 12 ч)
- `GET/POST /api/capsules`, `PUT/DELETE /api/capsules/:id`, `POST /api/capsules/:id/access` — капсулы
- `GET/POST /api/capsules/:id/contacts`, `POST /api/contacts/:id/revoke`, `DELETE /api/contacts/:id` — доверенные лица
- `GET /api/audit` — журнал владельца
- `POST /api/capsule/:uuid/request-code`, `POST /api/capsule/:uuid/verify`, `GET /api/capsule/:uuid/content` — публичный поток верификации

## Сознательно не в MVP

Автотриггер «мёртвой руки», шифрование файлов at rest (файлов пока нет — только текст и ссылка на видео), паспортная верификация, разные уровни доступа. HTTPS — на уровне деплоя (reverse proxy).
